# SkillDrive 条件 CVAE 基线阶段审查

更新日期：2026-07-21。

## 当前结论

04 Goal已经在修正后的v5监督合同上完成：500/100开发缓存、32样本过拟合、开发训练与独立评估、训练性能优化、20,000/2,000正式缓存、正式训练、独立内部验证以及16例BEV诊断。旧v2/v4缓存、checkpoint和指标均因监督语义变化失效，本文不沿用任何旧版结果。

正式模型在2,000条基础样本上的Prior `minADE@6/minFDE@6`为`2.2768/5.3063 m`，优于恒速基线`4.6136/11.9434 m`；在954条真实技能样本上为`0.7310/1.4760 m`，也优于恒速基线`1.3377/3.2739 m`。这证明当前条件CVAE基线和完整训练链路有效，但不等于34类技能均已学会稳定可控生成。

Final Validation在本阶段始终禁止访问；训练、选模、指标和诊断均只使用Formal Train与Internal Validation。当前全量`pytest`退出码为0，测试100%通过。

## 监督语义修复与v5失效边界

此前`short_headway_following`错误地把前车leader作为预测目标。修复后：

- `leader`必须映射到initiator，`close_follower`必须映射到responder；
- 技能监督目标固定为实际执行短时距跟驰行为的`close_follower`；
- 第50–109帧未来窗重新检查跟车速度、正向间距、时距和横向偏移；
- 只有连续至少1.0秒满足速度不低于2.0 m/s、时距不超过1.2秒、正向间距大于0且横向偏移不超过3.75 m的样本才保留。

这项修复会改变目标轨迹、局部坐标锚点和样本ID，因此缓存版本提升为v5。读取端会拒绝旧版本，开发与正式四个分区均已强制重建；旧v2/v4缓存、训练权重、评估和诊断只能视为历史中间产物，不能与v5结果混用。

监督口径同时保持以下限制：

- 97,664条正式候选记录中，`compatible_seed`只表示适合后续生成反事实，不作为技能正监督；正式Train中的83,518条`compatible_seed`均被忽略；
- 只有`detection_mode == "observed_trigger"`的记录可以产生技能监督样本；
- `sampled_parameters_json`是规则采样或接口参数，不是真实轨迹的连续参数标签；当前`parameter_mask`保持为0；
- 基础样本使用`skill_id=<none>`且`skill_supervision_mask=0`，真实技能样本才启用技能监督；
- Final Validation不得用于调参、选模、诊断或本文指标。

## 性能优化与冻结配置

有效候选均使用相同v5 Development Train缓存指纹、相同模型/损失、20步预热、200步测量、3次重复和固定采样合同。核心候选结果为：

| --- | --- | ---: | ---: |
| 16 / 4 | 否 | 120.15样本/s | 203.79 min |
| 128 / 8 | 否 | 820.73样本/s | 29.83 min |
| 512 / 0 | 否 | 1,562.49样本/s | 15.67 min |
| 512 / 4 | 否 | 1,722.28样本/s | 14.22 min |
| 512 / 8 | 否 | 1,726.54样本/s | 14.18 min |
| 640 / 0 | 否 | 1,610.73样本/s | 15.20 min |
| 640 / 4 | 否 | 1,781.74样本/s | 13.74 min |
| 640 / 8 | 否 | 1,920.20样本/s | 12.75 min |
| **672 / 8** | **是** | **1,926.85样本/s** | **12.71 min** |
| 688 / 8 | 否 | 1,919.14样本/s | 12.76 min |

最终冻结为`batch_size=672`、`num_workers=8`、`persistent_workers=true`、`prefetch_factor=2`、pin memory、BF16 AMP、TF32关闭，并使用WSL ext4运行时缓存。该候选三次吞吐范围为1,892.71–1,958.21样本/s，中位P50/P95步耗时为0.3463/0.3722秒，DataLoader等待占比0.217%，GPU平均利用率80.4%，峰值显存6,680.50 MiB。

按29,382条正式Train样本训练50 epoch计算，早期16/4基线约需12,227.2秒，冻结配置约需762.4秒，绝对节省11,464.8秒，即约3小时11分05秒；时间减少93.76%，吞吐约为16.04倍。相对此前512/0候选仍可再节省177.8秒，即约2分58秒，时间减少18.91%。

对672/8关闭persistent workers的额外复核中位吞吐为1,935.75样本/s，只比开启时高8.90样本/s；折算完整正式训练仅差3.5秒、相对0.46%，且落在重复波动内。正式训练保留persistent workers以避免每个epoch重建worker。688没有稳定增益，704出现明显吞吐回落，因此继续增大batch已无明显收益。配置冻结同时考虑绝对节时、相对变化和重复波动，不使用固定百分比门槛。

## 16例BEV诊断抽查

诊断程序固定从Internal Validation选择16例，包含8条基础样本和8条真实技能样本；每例绘制真实未来、Posterior重建和6条Prior轨迹，并保存完整样本、latent、轨迹、图片和checkpoint哈希。`diagnostics.json`明确记录`final_validation_accessed=false`。16/16张图片均成功生成且文件哈希完整。

人工视觉抽查采用“至少一条Prior大体保持真实行驶方向且无明显离路发散”作为定性标准，12/16例可接受，4/16例存在明显失败或不稳定。该数字只用于定位问题，不是随机样本上的技能成功率。

成功案例包括：

- 图08 `slow_lead_blockage`：Prior `minADE/minFDE=0.123/0.133 m`，停车趋势与真实未来一致；
- 图12 `lead_sudden_stop`：Prior `0.648/0.225 m`，至少一条生成轨迹正确覆盖短程制动；
- 图13、图14 `slow_lead_blockage`：Prior终点误差分别为0.118 m和0.462 m，生成轨迹保持合理方向。

失败或不稳定案例包括：

- 图04基础样本：Prior `minFDE=5.222 m`，最大端点分离29.720 m，多条轨迹在路口向错误分支发散；
- 图06基础样本：Prior `minFDE=14.456 m`、最大端点分离68.984 m，出现明显离路和方向错误；
- 图07基础样本：Posterior重建较好，但Prior `minFDE=7.095 m`，生成端点偏离真实短程未来；
- 图15 `short_headway_following`：Posterior能够跟随真实弯道，但Prior `minFDE=16.417 m`，多条生成轨迹未学会真实转弯。

这些失败说明总体平均指标改善并未消除复杂路口、短程运动和稀有技能上的Prior失配，后续阶段仍需针对技能平衡、地图约束和生成筛选继续改进。

## 已知限制

- 地图与碰撞辅助损失接口已经实现，但正式配置的`map_soft_weight`和`collision_soft_weight`均为0，因此本次正式指标不包含辅助损失收益；
- 当前地图损失只是预测点到地图折线支撑的软距离，不是严格的可行驶区域、车道拓扑或交通规则判定，不能表述为已经验证地图合规；
- `compatible_seed`没有作为正标签，`sampled_parameters_json`没有作为真实参数监督，当前结果不能证明连续参数控制能力；
- Posterior读取真实未来，只能作为重建诊断；可部署生成结论必须以Prior为准；
- Internal Validation仅覆盖实际发生并通过完整性过滤的技能，不能替代34类技能的均衡控制评估；
- Final Validation仍未访问，保留给后续最终一次评估。
